Task: decide if law A implies law B for all magmas (one binary op, no associativity unless forced).

Default FALSE. But do NOT say FALSE before seriously trying substitution. Most mistakes are false negatives.

Main method: substitution only.
By Birkhoff, A |= B iff every instance of B can be obtained by uniformly substituting terms for variables in A, possibly using either side of A as the source pattern. Try hard to match B to an instance of A.

How to try:
1) Compare shapes with full parentheses. Never reassociate.
2) Try both directions of A: lhs(A)=rhs(A) and rhs(A)=lhs(A).
3) Replace each variable of A by any term built from variables of B.
4) The same variable of A must be replaced consistently everywhere.
5) Different variables of A may map to the same term.
6) If after substitution the two sides become exactly B’s two sides, then TRUE.

Fast heuristics before/while matching:
- Variable count can only stay same or decrease under substitution, never increase.
- Repetition pattern matters: if one variable occurs twice in A, its image must appear in both corresponding places in B.
- A variable occurring on only one side of A can still map to a term with repeated variables.
- If B has fewer distinct variables, try identifying variables of A.
- If B has a constant-like behavior x or y repeated everywhere, try substituting variables of A by one variable.
- Left/right position and bracketing must match after substitution.

Useful special check:
If A is x*(y*z)=y or x*(y*z)=z or similar projection/identity-looking law, test substitutions that collapse variables (x:=y, y:=z, etc.). Many TRUE cases come from this.

Common wrong move:
Assuming “similar algebraic flavor” means implication. It does not.
Especially never assume associativity, commutativity, cancellation, identities, or idempotence unless derivable by substitution.

When substitution seems impossible:
Then usually FALSE. As a last resort, seek a 2-element magma counterexample satisfying A but not B. But do not spend long on tables; overusing counterexamples hurts accuracy.

Decision rule:
Exact substitution match found -> TRUE.
No match after trying both directions and obvious variable identifications/collapses -> FALSE.
