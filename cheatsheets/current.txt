Magma implication cheatsheet: does Eq1 => Eq2?
(every magma satisfying Eq1 also satisfies Eq2)

KEY DEFINITIONS:
- LM(T) = leftmost variable of T. Rule: LM(A*B)=LM(A), LM(var)=var.
- RM(T) = rightmost variable of T. Rule: RM(A*B)=RM(B), RM(var)=var.
- "n_other" = number of distinct variables in T other than x (for x=T forms).

================================================================
DECISION TREE (follow in exact order)
================================================================

NODE 1: Is Eq1 of form v = T (or T = v) with v absent from T?
  YES => Answer TRUE. This is absolute, zero exceptions.
         (v must be a single variable, not a compound term.)

NODE 2: Does a counterexample magma exist?
  Test LZ, RZ, C0, XOR (details below).
  If Eq1 holds in any magma but Eq2 fails => Answer FALSE.

NODE 3: Is Eq1 self-referential? (form x = T where x appears in T)
  YES => Go to NODE 3A.
  NO => If both sides of Eq1 are compound terms, lean FALSE.
        Otherwise lean FALSE.

NODE 3A: Self-reference boundary analysis
  Compute LM(T) and RM(T). Count variables in T other than x (n_other).

  If LM(T) = x AND RM(T) = x (both boundaries):
    => Answer FALSE (these are very weak equations).

  If n_other >= 2 (at least 2 variables besides x in T):
    => Answer TRUE (strong equation, forces most things).

  If n_other < 2 (only x and at most 1 other variable in T):
    => Answer FALSE (not strong enough to force arbitrary equations).

================================================================
COUNTEREXAMPLE MAGMA DETAILS (for Node 2)
================================================================

For each magma: first check Eq1 holds, then check Eq2 fails.
If Eq1 does NOT hold in a magma, that magma cannot serve as counterexample.

--- Left-Zero (LZ): a*b = a ---
  Every term evaluates to its leftmost variable.
  Equation holds in LZ iff LM(left side) = LM(right side).

--- Right-Zero (RZ): a*b = b ---
  Every term evaluates to its rightmost variable.
  Equation holds in RZ iff RM(left side) = RM(right side).

--- Constant-Zero (C0): a*b = 0 on {0,1} ---
  Any compound term (with *) becomes 0. Lone variable stays itself.
  Equation holds iff both sides are the same lone variable,
  or both sides contain * (both become 0).

--- XOR: a*b = (a+b) mod 2 on {0,1} ---
  Term value = sum of all variable occurrences mod 2.
  Equation holds iff each variable appears the same number of
  times (mod 2) on both sides.

================================================================
HOW TO COMPUTE LM AND RM (critical skill)
================================================================

LM: always follow the LEFT branch at each * node.
RM: always follow the RIGHT branch at each * node.

LM(x) = x                    RM(x) = x
LM(A*B) = LM(A)              RM(A*B) = RM(B)

Examples:
  ((y*x)*z)*w:   LM=y  RM=w
  y*(z*(w*(y*z))): LM=y  RM=z
  (x*(y*x))*((y*y)*z): LM=x  RM=z
  x*(y*(z*w)):   LM=x  RM=w

================================================================
HOW TO COUNT n_other
================================================================

For Eq1 of form x = T, count how many DISTINCT variables appear
in T that are NOT x.

Examples:
  x = y*(z*(w*(y*z)))  => variables in T: {y,z,w} => n_other=3
  x = y*(x*(x*y))      => variables in T: {x,y} - {x} = {y} => n_other=1
  x = (y*x)*(x*(x*z))  => variables in T: {x,y,z} - {x} = {y,z} => n_other=2
  x = x*(x*(x*x))      => variables in T: {x} - {x} = {} => n_other=0

================================================================
WORKED EXAMPLES
================================================================

EXAMPLE 1: TRUE via Node 1 (lone variable absent)
  Eq1: x = y*(z*(w*(y*z)))   Eq2: x = ((x*(x*y))*y)*x
  Variables in T: y, z, w. Variable x is absent.
  => Node 1 fires. Answer: TRUE.

EXAMPLE 2: FALSE via Node 2 (LZ counterexample)
  Eq1: x = x*((y*z)*w)   Eq2: x = y*(z*(x*w))
  LZ check: LM(x)=x, LM(x*((y*z)*w))=x. Eq1 holds in LZ.
  LM(x)=x, LM(y*(z*(x*w)))=y. x != y. Eq2 fails in LZ.
  => Node 2 fires. Answer: FALSE.

EXAMPLE 3: TRUE via Node 3A (strong self-ref)
  Eq1: x = y*((z*z)*(x*w))   Eq2: x*y = (z*(w*z))*y
  Node 1: x in T, does not apply.
  Node 2: LZ: LM(x)=x, LM(y*...)=y, x!=y. Eq1 fails in LZ.
          RZ: RM(x)=x, RM(...*(x*w))=w, x!=w. Eq1 fails in RZ.
          No counterexample found.
  Node 3: Self-ref. T=y*((z*z)*(x*w)).
          LM(T)=y, RM(T)=w. Neither is x.
          Variables in T besides x: {y,z,w}. n_other=3.
          n_other >= 2 => Answer: TRUE.

EXAMPLE 4: FALSE via Node 3A (weak self-ref, few other vars)
  Eq1: x = (y*(y*(x*y)))*y   Eq2: x*y = (y*x)*(z*z)
  Node 1: x in T, does not apply.
  Node 2: No LZ/RZ counterexample (check shows Eq1 fails in both).
  Node 3: Self-ref. T=(y*(y*(x*y)))*y.
          LM(T)=y, RM(T)=y. Neither is x. Good.
          Variables in T besides x: {y}. n_other=1.
          n_other < 2 => Answer: FALSE.

EXAMPLE 5: Avoiding common LZ/RZ mistake
  Eq1: x = ((y*x)*y)*(z*w)   Eq2: x*(y*z) = (y*z)*y
  LZ: LM(x)=x, LM(((y*x)*y)*(z*w))=LM((y*x)*y)=LM(y*x)=y.
  x != y => Eq1 does NOT hold in LZ. Cannot use LZ.
  RZ: RM(x)=x, RM(((y*x)*y)*(z*w))=RM(z*w)=w.
  x != w => Eq1 does NOT hold in RZ. Cannot use RZ.
  Self-ref: LM(T)=y!=x, RM(T)=w!=x. n_other=3 (y,z,w).
  n_other >= 2 => Answer: TRUE.

================================================================
QUICK REFERENCE
================================================================
v=T, v absent from T       => ALWAYS TRUE
counterexample magma found  => ALWAYS FALSE
self-ref, both boundary     => FALSE
self-ref, n_other >= 2      => TRUE  (high confidence)
self-ref, n_other < 2       => FALSE (high confidence)
composite = composite       => lean FALSE
uncertain                   => lean FALSE
LM(A*B) = LM(A).  RM(A*B) = RM(B).
