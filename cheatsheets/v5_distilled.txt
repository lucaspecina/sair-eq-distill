MAGMA EQUATION IMPLICATION CHEAT SHEET  
Goal: decide whether equation A ⇒ equation B (i.e., every magma satisfying A must also satisfy B).

A magma is a set with a binary operation *.  
An equation is a universally quantified identity over variables (e.g., x*(y*z) = (x*y)*z).

IMPORTANT FACTS
- 37% TRUE, 63% FALSE → default guess is FALSE unless you can derive B.
- Implication is transitive: if A ⇒ C and C ⇒ B then A ⇒ B.
- One strongest law: x = y (implies everything).
- One weakest law: x = x (implied by everything).
- Missing TRUE implications is the most common error.

---------------------------------------------------------------------

STEP 0: NORMALIZE
Treat equations as universally quantified.  
Variables can be renamed freely.

Example:  
x*(y*z) = y is equivalent to a*(b*c) = b.

---------------------------------------------------------------------

STEP 1: QUICK VARIABLE HEURISTICS

These often settle the problem instantly.

1. Variable count rule
If B uses MORE distinct variables than A, A usually CANNOT imply B.

Example:
A: x*y = x  (variables x,y)
B: x*y = z  (x,y,z)

A cannot force existence of new variable z → likely FALSE.

2. RHS-only variables (very strong laws)

If a variable appears on RHS but not LHS, the equation is extremely strong.

Example:
x*y = z

This forces all products to equal arbitrary elements → near x=y strength.

Such equations usually imply many others.

3. Variable disappearance

Equations where variables disappear are strong.

Examples:
x*y = x
x*y = y
x*y = c-type behavior

These often force projection or constant operations.

4. Same variable both sides

Weak examples:
x = x
x*y = x*y

These imply almost nothing.

---------------------------------------------------------------------

STEP 2: DIRECT SUBSTITUTION (MOST IMPORTANT)

Most TRUE implications are proved by substitution.

Method:
1. Start with equation A.
2. Substitute expressions for variables.
3. Use A repeatedly until B appears.

Example pattern:

A: x*(y*z) = y

Substitute:
y → x

Then:
x*(x*z) = x

Substitute:
z → y

x*(x*y) = x

Often B appears after 1–3 substitutions.

Key idea: since A holds for ALL variables, you may replace variables with any term.

Allowed substitutions:
x → y*z
y → x*x
etc.

---------------------------------------------------------------------

STEP 3: DETECT SPECIAL OPERATION TYPES

Many equations force the magma to have a very restricted form.

1. Left projection

x*y = x

Then every product equals the left input.

Consequences:
x*(y*z) = x
(x*y)*z = x
Many equations become automatically true.

2. Right projection

x*y = y

Then:
x*(y*z) = z
(x*y)*z = z

3. Constant operation

If A forces all products equal the same element c:

x*y = c

Then every term evaluates to c.

Therefore any equation with identical constant result holds.

Test for this by checking if A lets you show:
x*y = u*v

for arbitrary variables.

4. Left identity collapse

Sometimes A implies every element behaves like a left identity.

Example reasoning pattern:

1. Show some element e satisfies e*x = x.
2. Use A to show any element acts like e.
3. Conclude x*y = y.

Then the magma becomes right projection.

---------------------------------------------------------------------

STEP 4: REWRITE BOTH SIDES OF B

To prove A ⇒ B:

Try to rewrite BOTH sides of B using A until they match.

Example:

B: (x*y)*z = x

If A gives:
(x*y)*z = y
and also
y = x

then B holds.

Always attempt:
- substitute into A
- apply A inside larger expressions

---------------------------------------------------------------------

STEP 5: TRANSITIVITY TRICK

Sometimes A does not look like B, but A can produce an intermediate law C.

Example pattern:

A ⇒ x*y = x  
x*y = x ⇒ B

Then A ⇒ B.

Common intermediate laws:
x*y = x  
x*y = y  
x = y  
x*(y*z) = x*z  
(x*y)*z = x*z

---------------------------------------------------------------------

STEP 6: IF TRUE PROOF FAILS → SEARCH FOR COUNTEREXAMPLE

To prove FALSE, find a magma satisfying A but violating B.

Small magmas usually work.

Try 2-element magma {0,1}.

A magma is defined by a table:

* | 0 1
-------
0 | a b
1 | c d

where each entry is 0 or 1.

Useful tables (often good counterexamples):

Table 0001
* | 0 1
-------
0 | 0 0
1 | 0 1

Table 1000
* | 0 1
-------
0 | 1 0
1 | 0 0

These satisfy very few equations (~3%).

Procedure:

1. Check if table satisfies A.
2. If yes, test B.
3. If B fails → implication FALSE.

---------------------------------------------------------------------

STEP 7: FAST STRUCTURAL CHECKS

These patterns frequently decide cases.

1. If A forces x=y → TRUE.

Because x=y implies every equation.

2. If B is x=x → TRUE.

Everything implies x=x.

3. If A and B are syntactically identical up to renaming → TRUE.

4. If B obviously stronger than A → FALSE.

Example:

A: x*y = x  
B: x = y

Projection does not force all elements equal.

---------------------------------------------------------------------

COMMON TRUE-PROOF PATTERNS

Pattern 1: self-substitution

From:
A: x*(y*z) = y

Substitute y → x*(u*v) etc.

Pattern 2: variable identification

Set variables equal.

Example:
y = x

Pattern 3: nesting

Apply A inside subterms.

Example:
(x*(y*z))*w

---------------------------------------------------------------------

COMMON MISTAKES (AVOID)

1. Declaring FALSE too quickly.

TRUE implications are often hidden and require substitutions.

2. Trying counterexamples before attempting substitution.

First try to derive B from A.

3. Forgetting variables are universally quantified.

You may replace variables with ANY term.

4. Assuming associativity or other properties not given.

Only use equation A.

---------------------------------------------------------------------

PRACTICAL DECISION PROCEDURE

1. Normalize variable names.

2. Quick checks:
   - B is x=x → TRUE
   - A is x=y → TRUE
   - B uses more variables → likely FALSE

3. Attempt substitution proof:
   - substitute variables in A
   - try to rewrite sides of B

4. Look for forced operation types:
   - x*y=x
   - x*y=y
   - constant operation

5. If B still not derived → try 2-element counterexample tables.

6. If a table satisfies A but not B → FALSE.

---------------------------------------------------------------------

FINAL ADVICE

Because FALSE is more common (63%), but the main failure is missing TRUE proofs:

Always attempt several substitutions from A before concluding FALSE.
