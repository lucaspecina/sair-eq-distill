Magma implication cheatsheet (`*` binary, no axioms)

Goal: predict whether `Eq1 ⇒ Eq2` holds in all magmas satisfying `Eq1`.

Notation
- `Vars(E)`: variables in equation `E`
- `LM(T)`, `RM(T)`: leftmost / rightmost variable of term `T`
- lone-variable side: one whole side is exactly a variable `v`
- composite: any non-variable term
- XOR signature: parity (mod 2) of each variable’s occurrences in a term

==================================================
ABSOLUTE RULES
==================================================

1. LONE VARIABLE ABSENT = TRUE
- If Eq1 is `v = T` or `T = v` and `v` does NOT occur in `T`,
  then `Eq1 ⇒ Eq2` for EVERY Eq2.
- Stop immediately: TRUE.

2. COUNTEREXAMPLE MAGMAS = FALSE
- If Eq1 holds but Eq2 fails in ANY magma below, implication is FALSE.
- These checks had zero false positives on training data.

a) Left-zero (LZ): `a*b=a`
- Every term evaluates to `LM(term)`.
- `A=B` holds iff `LM(A)=LM(B)`.
- So FALSE if Eq1 has same leftmost variable on both sides, but Eq2 does not.

b) Right-zero (RZ): `a*b=b`
- Every term evaluates to `RM(term)`.
- `A=B` holds iff `RM(A)=RM(B)`.
- So FALSE if Eq1 has same rightmost variable on both sides, but Eq2 does not.

c) Constant-zero (C0) on `{0,1}`: `a*b=0`
- Every composite term evaluates to `0`; a lone variable stays that variable.
- Universal truth table in C0:
  - composite = composite : always true
  - `v = composite` / `composite = v` : not universally true
  - `u = v` : true iff same variable symbol
- So if Eq1 is universally true in C0 but Eq2 is not, then FALSE.

d) XOR on `{0,1}`: `a*b = a+b mod 2`
- Every term = parity sum of its variables.
- `A=B` holds iff each variable has same parity on both sides.
- If Eq1 parity matches but Eq2 parity does not, then FALSE.

==================================================
SAFE TRUE RULES
==================================================

3. DIRECT SUBSTITUTION INSTANCE = TRUE
- If Eq2 is exactly a uniform substitution instance of Eq1, then TRUE.
- Renaming variables is included.
- Same source variable must be replaced consistently everywhere.

4. DIRECT TRANSPORT FROM SELF-REFERENCE = TRUE
- If Eq1 is `x = T(x,y1,...,yk)` or `T(x,y1,...,yk) = x`,
  then for any term `P`, substituting `x:=P` gives
  `P = T(P,y1,...,yk)` or `T(P,y1,...,yk)=P`.
- If Eq2 is exactly of this form, possibly with a further uniform substitution
  on the other variables, then TRUE.
- The SAME term `P` must replace every occurrence of `x`.

5. EXPLICIT SHORT CHAIN = TRUE
- Multiple uses of Eq1 are allowed only when you can write a short literal chain,
  each step being exactly Eq1 or its symmetric form under substitution.
- If no explicit chain is visible, do NOT claim TRUE from “similar shape”.

==================================================
VERY IMPORTANT DERIVED TRUE RULE
==================================================

6. RHS/LHS-INDEPENDENT COLLAPSE
- If Eq1 is `x = T` or `T = x` and there exists a variable `a` with:
  - `a` occurs in `T`, and
  - `a` does NOT occur anywhere in Eq2,
- then Eq1 implies every variable is equal in any model:
  choose two valuations differing only in `a`; LHS stays `x`, so `T` must be
  independent of `a`; hence all values of `x` are equal. Therefore every Eq2 holds.
- Practical shortcut: if Eq1 has a lone variable side and Eq2 omits ANY variable
  appearing on the opposite side of Eq1, then TRUE.
- This correctly covers many non-Rule-1 TRUE cases.
Examples:
- `x = ((y*z)*w)*(u*v)  ⇒  any Eq2 not using at least one of {y,z,w,u,v}` is TRUE.
- `x = ((y*z)*(w*x))*u  ⇒  any Eq2 omitting `u` is TRUE.

==================================================
FALSE-LEANING FACTS
==================================================

7. NO LONE-VARIABLE SIDE IN Eq1 ⇒ LEAN FALSE
- If Eq1 is composite = composite, implication is usually FALSE
  unless Rule 3 or Rule 5 gives a concrete proof.

8. 2+ FRESH VARIABLES IN Eq2 ⇒ STRONG FALSE LEAN
- If Eq2 uses at least 2 variables not in Eq1, prefer FALSE,
  unless Rule 1, 3, 4, 5, or 6 applies.

9. SELF-REFERENCE ALONE IS NOT A PROOF
- Eq1 of form `x = T(x,...)` does NOT by itself make other `x = U(x,...)` laws true.
- Do not infer TRUE from “same absorbing/self-referential pattern”.

10. REMOVE THE OLD “MISSING BOUNDARY ⇒ PREFER TRUE” HEURISTIC
- This caused false positives.
- If Eq1 is `x = T(x,...)` with `LM(T)≠x` or `RM(T)≠x`, that ONLY means Eq1 fails
  in LZ or RZ respectively, so that magma cannot be used as a counterexample.
- It does NOT make Eq2 likely true by itself.

==================================================
ANTI-MISTAKE WARNINGS
==================================================

11. RULE 1 DEPENDS ONLY ON Eq1
- Never apply “lone variable absent” to Eq2.

12. CHECK WHETHER Eq1 HOLDS IN THE SAME TEST MAGMA
- For LZ/RZ, if Eq1’s boundary variables differ, Eq1 fails there, so that magma is unusable.
- Common mistake: showing Eq2 fails in LZ/RZ without first checking Eq1 holds there.

13. C0 IS UNIVERSAL-VALIDITY, NOT ONE VALUATION
- Need Eq1 true for all valuations in C0, not just some.
- Fast classification:
  - composite=composite : universally true
  - variable=composite or composite=variable : not universally true
  - `u=v` : universally true iff same symbol

14. XOR IGNORES ORDER AND BRACKETING
- Only variable parities matter.

15. SHAPE SIMILARITY IS NOT DERIVABILITY
- Shared contexts/subterms are irrelevant without substitution or explicit chain.

==================================================
DECISION PROCEDURE
==================================================

1. Parse Eq1/Eq2 and compute:
   - lone-variable sides
   - variable sets
   - `LM`, `RM`
   - XOR signatures

2. Apply Rule 1.
   - If Eq1 is `v=T` or `T=v` with `v` absent from `T` ⇒ TRUE.

3. Apply Rule 6.
   - If Eq1 has lone-variable side `x` and Eq2 omits some variable occurring on
     the opposite side of Eq1 ⇒ TRUE.

4. Apply Rule 2 countermodels.
   - LZ, RZ, C0, XOR.
   - If any gives Eq1 true and Eq2 false ⇒ FALSE.

5. Check safe proofs.
   - direct substitution instance ⇒ TRUE
   - direct self-reference transport ⇒ TRUE
   - explicit short chain ⇒ TRUE

6. Default.
   - If Eq1 has no lone-variable side ⇒ FALSE.
   - If Eq2 has 2+ fresh variables ⇒ FALSE.
   - Otherwise prefer FALSE unless a concrete TRUE proof exists.

==================================================
COMPACT REMINDERS
==================================================

- `x = T` with no `x` in `T` => ALWAYS TRUE.
- If Eq1 has lone-variable side and Eq2 omits any variable from the other side => TRUE.
- LZ: term = leftmost variable.
- RZ: term = rightmost variable.
- C0: composite terms all become `0`.
- XOR: compare parity only.
- Never use LZ/RZ unless Eq1 itself holds there.
- Do NOT use “same self-referential pattern” as proof.
- When uncertain after all checks, prefer FALSE.
