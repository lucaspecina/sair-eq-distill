Magma implication cheatsheet (`*` binary, no axioms)

Goal: predict whether `Eq1 ⇒ Eq2`, meaning every magma satisfying `Eq1` also satisfies `Eq2`.

Notation
- `Vars(E)`: variables in equation `E`
- `LM(T)`: leftmost variable in term `T`
- `RM(T)`: rightmost variable in term `T`
- “lone-variable side”: one whole side is exactly a variable `v`
- “composite”: any non-variable term
- XOR parity signature: for each variable, count occurrences mod 2

==================================================
FIRST: TWO PERFECT RULES
==================================================

1. LONE VARIABLE ABSENT = TRUE
- If Eq1 is `v = T` or `T = v` and `v` does NOT occur anywhere in `T`,
  then `Eq1 ⇒ Eq2` for EVERY Eq2.
- Absolute rule. Depends only on Eq1. Stop: TRUE.

2. COUNTEREXAMPLE MAGMAS = FALSE
- If Eq1 holds but Eq2 fails in ANY one of the 4 magmas below, then implication is FALSE.
- These give zero false positives on training data.

a) Left-zero (LZ): `a*b=a`
- Every term evaluates to `LM(term)`.
- `A=B` holds iff `LM(A)=LM(B)`.
- So if `LM(Eq1L)=LM(Eq1R)` but `LM(Eq2L)≠LM(Eq2R)`, then FALSE.

b) Right-zero (RZ): `a*b=b`
- Every term evaluates to `RM(term)`.
- `A=B` holds iff `RM(A)=RM(B)`.
- So if `RM(Eq1L)=RM(Eq1R)` but `RM(Eq2L)≠RM(Eq2R)`, then FALSE.

c) Constant-zero (C0) on `{0,1}`: `a*b=0`
- Every composite term evaluates to `0`.
- A lone variable stays that variable.
- Thus:
  - composite = composite : always true in C0
  - `v = composite` or `composite = v` : equivalent to `v=0`, not universally true
  - `u = v` : equivalent to `u=v`, not universally true
- If Eq1 is true in C0 under all valuations but Eq2 is not, then FALSE.
- Fast tests:
  - If Eq1 is composite=composite, it holds in C0.
  - If Eq2 is variable=composite, composite=variable, or variable=variable with distinct vars, C0 may kill it.

d) XOR on `{0,1}`: `a*b = a+b mod 2`
- Each term evaluates to parity sum of its variables.
- `A=B` holds iff every variable has the same parity on both sides.
- If Eq1 parity matches but Eq2 parity does not, then FALSE.

==================================================
SAFE TRUE RULES
==================================================

3. DIRECT SUBSTITUTION INSTANCE = TRUE
- If Eq2 is exactly a uniform substitution instance of Eq1, then TRUE.
- Renaming variables is a special case.
- Same source variable must be replaced by the same term everywhere.

4. DIRECT TRANSPORT FROM SELF-REFERENCE = TRUE
- If Eq1 is `x = T(x, y1,...,yk)` or `T(x, y1,...,yk) = x`,
  then for any term `P`, substituting `x:=P` yields
  `P = T(P, y1,...,yk)` or `T(P, y1,...,yk) = P`.
- If Eq2 is exactly of that form, possibly with a further uniform substitution
  on the other variables, then TRUE.
- Important: the SAME term `P` must replace every occurrence of `x`.

5. EXPLICIT EQUALITY CHAIN = TRUE
- Multiple uses of Eq1 are allowed only if you can write a short literal chain:
  each step is exactly a substitution instance of Eq1 or its symmetric form.
- If no clean chain is visible, do NOT claim TRUE.

==================================================
IMPORTANT HIGH-PAYOFF FACT
==================================================

6. SELF-REFERENCE WITH A MISSING BOUNDARY IN Eq1 IS VERY STRONG
- Pattern: Eq1 is `x = T(...x...)` or `T(...x...) = x`, and `x` occurs in `T`,
  BUT `LM(T) ≠ x` OR `RM(T) ≠ x`.
- Then Eq1 survives at least one of LZ/RZ automatically:
  - if `LM(T) ≠ x`, Eq1 fails in LZ
  - if `RM(T) ≠ x`, Eq1 fails in RZ
- So NEVER use that failing magma as a counterexample.
- In practice, such Eq1 is often very strong. After all Rule-2 checks fail,
  if Eq2 is also of the form `x = U(...x...)` or `U(...x...) = x`, prefer TRUE.
- This is a heuristic, not a proof. Do not override an actual counterexample.

Useful examples this rescues:
- `x = (y * (x * (z * z))) * z`
- `x = (y * x) * (y * (z * y))`

==================================================
FALSE-LEANING PATTERNS
==================================================

7. NO LONE-VARIABLE SIDE IN Eq1 ⇒ LEAN FALSE
- If Eq1 is composite = composite, implication is usually FALSE
  unless Eq2 is a direct substitution instance or explicit chain consequence.

8. MANY FRESH VARIABLES IN Eq2 ⇒ STRONG FALSE LEAN
- If Eq2 has 2 or more variables not in Eq1, prefer FALSE,
  unless Rule 1 or Rule 3/4/5 applies.

9. SELF-REFERENCE ALONE IS NOT ENOUGH
- If Eq1 is `x = T(x,...)` and Eq2 is not a direct transport/substitution/chain consequence,
  do NOT auto-say TRUE.
- Especially lean FALSE when Eq2 changes the surrounding context completely.

10. BOUNDARY KILL TESTS FOR Eq2
- If `LM(Eq2L) ≠ LM(Eq2R)`, then Eq2 fails in LZ.
- If `RM(Eq2L) ≠ RM(Eq2R)`, then Eq2 fails in RZ.
- But this only gives FALSE if Eq1 holds in that same magma.
- Always check Eq1 first in the same magma.

==================================================
ANTI-MISTAKE WARNINGS
==================================================

11. RULE 1 DEPENDS ONLY ON Eq1
- Never apply “lone variable absent” to Eq2.

12. DO NOT MISREAD RZ/LZ ON SELF-REFERENTIAL Eq1
- In RZ, `x = T` becomes `x = RM(T)`.
- If `RM(T) ≠ x`, then Eq1 FAILS in RZ, so RZ cannot be a counterexample.
- In LZ, `x = T` becomes `x = LM(T)`.
- If `LM(T) ≠ x`, then Eq1 FAILS in LZ, so LZ cannot be a counterexample.
- This exact mistake causes false negatives.

13. SHARED SHAPE IS NOT A PROOF
- Similar-looking terms or shared subterms do not imply derivability.
- Need uniform substitution, direct transport, or explicit chain.

14. C0 CHECK CAREFULLY
- composite=composite always holds in C0.
- variable vs composite usually fails.
- variable=variable holds only when the two variables are the same symbol.

15. XOR = PURE PARITY
- Ignore bracketing and order; count each variable mod 2 on both sides.

==================================================
DECISION PROCEDURE
==================================================

1. Parse Eq1/Eq2. Compute:
   - lone-variable sides
   - `LM`, `RM`
   - variable sets
   - XOR parity signatures
   - whether Eq1 is self-referential `x = T(x,...)` or `T(...,x)=x`

2. Apply Rule 1.
   - If Eq1 is `v = T` or `T = v` with `v` absent from `T` ⇒ TRUE.

3. Apply Rule 2 countermodels carefully.
   - LZ: only if Eq1 holds there.
   - RZ: only if Eq1 holds there.
   - C0: use variable/composite distinctions.
   - XOR: compare parity signatures.
   - If any counterexample exists ⇒ FALSE.

4. Check safe TRUE proofs.
   - direct substitution instance ⇒ TRUE
   - direct transport from self-reference ⇒ TRUE
   - short explicit equality chain ⇒ TRUE

5. Use strong heuristic for self-reference with missing boundary.
   - If Eq1 is `x = T(x,...)` or `T(...,x)=x`
   - and `x` is missing from at least one boundary of `T`
   - and all 4 countermodel checks failed
   - and Eq2 is also an `x`-absorbing style law (`x = U(...x...)` or symmetric),
     then prefer TRUE.

6. Otherwise default:
   - If Eq1 has no lone-variable side ⇒ FALSE.
   - If Eq2 has 2+ fresh variables ⇒ FALSE.
   - Else prefer FALSE unless you have a concrete derivation.

Compact reminders
- `x = T` with no `x` in `T` => ALWAYS TRUE.
- LZ checks leftmost vars only.
- RZ checks rightmost vars only.
- If Eq1 fails in LZ/RZ, that magma is unusable as counterexample.
- C0: every composite is `0`.
- XOR: parity match required.
- Direct substitution / direct transport / explicit chain => TRUE.
- When uncertain after checks, prefer FALSE.
