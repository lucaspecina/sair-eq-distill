Decoder
- Law / identity: universally quantified equation between binary terms built from variables and one operation *.
- Magma: arbitrary set with one binary operation; assume no associativity, cancellation, etc.
- “Eq1 ⇒ Eq2” means: every magma satisfying Eq1 also satisfies Eq2.
- Allowed moves: rename variables, substitute variables by arbitrary terms, instantiate repeatedly, and chain equalities.
- Key method: if one law lets you prove the magma is one-element (all terms equal), then every Eq2 follows.

Core explosive rule
1. TOTAL COLLAPSE from a lone variable with no self-occurrence:
- If Eq1 has form v = T and v does NOT occur in T, then Eq1 implies every law.
- Proof schema: for any terms P,Q, instantiate v:=P and parameters in T so RHS becomes one common term R; then P=R and Q=R, hence P=Q.
- Symmetric if T = v with v absent from T.
- IMPORTANT: this is much stronger than the old “fixed-point” cases. If the lone-variable side variable is absent from the other side, it is ALWAYS explosive.

Super-important correction
- Do NOT hesitate when Eq1 is x = complicated_term_not_containing_x.
- That is already enough for total collapse, even if:
  - the RHS has many other variables,
  - the RHS is very deep,
  - Eq2 looks unrelated.
- Typical TRUE pattern:
  - x = y*(y*(z*(y*y)))
  - x = (y*(z*(z*y)))*w
  - x = y*(y*((y*y)*y))
These all force one-element magmas.

How to use the collapse rule concretely
- If Eq1 is x = T(y,z,w,...) with x absent from T:
  - For any variables a,b, instantiate x:=a with the same chosen values of y,z,w,... ; get a = R.
  - Instantiate x:=b with the same chosen values; get b = R.
  - Hence a=b.
- Once all variables are equal, every identity holds.

Direct consequence patterns
2. Direct substitution / renaming
- If Eq2 is a substitution instance or variable-renaming of Eq1, then TRUE.
- Also TRUE if one or two obvious substitution+transitivity steps derive Eq2.

3. Fixed-point transport
- From x = F(x,params), infer P = F(P,params) for any term P by substituting x:=P.
- Useful only if Eq2 has exactly that transported shape.

Safe synchronization
4. Synchronize only by literally making one side identical
- From two instances of Eq1, if you can force the RHSs to be the same term under chosen substitutions, then the LHSs are equal.
- Or if you can force the LHSs identical, then the RHSs are equal.
- This is the main way self-referential laws can still become strong.

New strong self-reference pattern to remember
5. If Eq1 is x = A*(x*B) or x = A*(B*(x*C)) etc., sometimes substitute for x using Eq1 itself and then compare with another instance to eliminate the inner x. Do this only when an explicit chain exists.
- But if x is absent from RHS, no such work is needed: collapse is immediate.

Main anti-overgeneralization rules
6. If neither side is a lone variable, default FALSE unless Eq2 is a clear instance/consequence.
- Composite = composite laws are usually weak.

7. If Eq1 is x = T(x,...) with x present on RHS, that is NOT automatic collapse.
- Examples: x = x*U, x = U*x, x = (x*B)*w, x = y*(y*(x*z)), x = (y*y)*(x*(y*z)).
- Treat as rewrite/fixed-point only unless you can derive a variable-free-of-x term equal to x, or synchronize explicitly.

8. “A variable disappears from one side” is weak unless the whole opposite side is a lone variable absent from the other side.
- Bad inference: from x*y = R(x,z) conclude all products x*y are equal. Usually false.

9. Balanced equations with real structure on both sides usually do NOT imply arbitrary lone-variable targets.

Failure-pattern fixes from past mistakes
10. TRUE cases the model missed:
- x*y = y*((z*w)*w)  can still imply strong laws if Eq2 is obtained by a direct substitution / synchronization. Do not auto-reject composite-left laws; check for easy instance first.
- Any Eq1 of form x = T with x absent from T is explosive. This includes:
  - x = (y*(z*(z*y)))*w
  - x = y*(y*(z*(y*y)))
  - x = y*(y*((y*y)*y))
- Do NOT label these “not explosive” just because RHS is composite.

11. Self-referential TRUE cases are harder:
- x = y*(((x*y)*x)*z)
- x = y*(y*(x*z))
- x = (y*y)*(x*(y*z))
- x = x*(((y*z)*z)*z)
These are NOT automatically TRUE/FALSE. You must try:
  a) substitute Eq1 into the inner x,
  b) instantiate with special replacements (x:=y, y:=x, etc.),
  c) synchronize repeated RHS/LHS terms,
  d) see whether Eq2 is a transported instance.
- But absent a concrete derivation, still lean FALSE.

Model-finding heuristics for FALSE
12. Try tiny magmas:
- Left-zero: a*b=a
- Right-zero: a*b=b
- Constant: a*b=c
Quick term evaluation:
- In left-zero, every term = its leftmost variable.
- In right-zero, every term = its rightmost variable.
If Eq1 holds but Eq2 fails in one, implication is FALSE.

Decision checklist
1. Does Eq1 have a lone variable side v?
   - No: usually FALSE unless Eq2 is an obvious instance/consequence.
   - Yes: continue.
2. Does v occur on the other side?
   - No: TOTAL COLLAPSE ⇒ TRUE for every Eq2.
   - Yes: only fixed-point/rewrite so far; continue cautiously.
3. Is Eq2 a direct substitution/renaming/one-step consequence of Eq1?
   - Yes: TRUE.
4. Can two instances be explicitly synchronized to force identical RHS or LHS?
   - Yes: use that.
5. Otherwise, especially for balanced or self-referential laws, prefer FALSE.

Bottom line
- Most important rule: lone variable = term not containing that variable ⇒ one-element magma ⇒ every target law TRUE.
- Keep caution only for x = T(x,...) or composite/composite laws.
- Don’t underuse the explosive rule; don’t overuse weaker “missing variable” patterns.
